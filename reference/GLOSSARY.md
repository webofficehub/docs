# Glossary

## A

**Absolute drain**  
A drain specified as a fixed amount of vector value.

**AuthRatio**  
A composite score that measures whether a vector is valid for a given operation.

## C

**Certified vector**  
A vector whose AuthRatio meets or exceeds the required threshold.

**Canonical ordering**  
The stable, protocol-defined order of vector components.

**Component**  
A single coordinate or token dimension within a vector.

## D

**Drain**  
A protocol-defined reduction or fee applied before or during an operation.

## F

**Free value**  
The portion of a vector not currently locked by projection or another hold state.

## M

**Magnitude**  
The aggregate value of a vector under the active valuation rule.

**Metadata**  
Descriptive fields that accompany a vector or wallet without changing the underlying vector components.

## O

**Origin Engine**  
The rule subsystem that authorizes new vector creation.

**Ownership proof**  
Cryptographic evidence that the initiator is entitled to act on the wallet.

## P

**Projection**  
The act of locking a defined portion of a vector into a controlled environment for later settlement.

**Proof**  
A cryptographic or procedural attestation associated with a state transition.

## R

**Record**  
An immutable event entry describing a state transition.

**Reconstruction**  
The return of a projected value after settlement.

## S

**Settlement**  
The process by which projected value is resolved into a final outcome.

**Space**  
A state environment in which vectors are stored and processed.

## T

**Transfer**  
The movement of value between wallets or spaces.

**Type**  
Metadata that defines how a vector may be interpreted and transformed.

## V

**Vector**  
A multidimensional state object composed of component values.

**Vector Network**  
The complete protocol system governing vectors, wallets, spaces, rules, and records.

**vWallet**  
An owner-bound container for vector state.
